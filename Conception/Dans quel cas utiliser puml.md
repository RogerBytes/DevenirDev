# Dans quel cas utiliser puml

Voici les principaux types de diagrammes PlantUML

- Use case diagrams (cas d’utilisation : acteurs, système, include/extend)
- Activity diagrams (processus métier : workflow, décisions, parallélisme)
- Class diagrams (structure objet : classes, héritage, relations)
- Deployment diagrams (infrastructure : serveurs, conteneurs, réseaux)
- Sequence diagrams (interactions temporelles entre systèmes/objets)

Voici des plus secondaires

- Class diagrams (structure objet : classes, héritage, relations)
- State diagrams (machines à états : transitions d’un objet selon événements)
- Component diagrams (architecture logicielle : modules et dépendances)
